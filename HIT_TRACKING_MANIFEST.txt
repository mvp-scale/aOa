KEY FINDINGS

WHAT EXISTS (WORKING)

✅ Three independent hit tracking systems:
   1. Domain/Term/Keyword hits (persistent, decayed)
   2. Prediction hits (rolling 24h, Hit@5 ~82%)
   3. Orphan hits (pre-domain tracking)

✅ Four hit increment functions:
   • increment_domain_hits()
   • increment_term_hits()
   • increment_keyword_hits()
   • increment_orphan_hits()

✅ Three trigger paths:
   1. Grep enrichment (primary, async background thread)
   2. Intent tag matching (secondary, synchronous)
   3. Prediction checking (tertiary, synchronous)

✅ Complete Redis storage structure:
   • Domain metadata hashes (with hits + total_hits)
   • Term/keyword/orphan hit hashes
   • Prediction rolling data (ZSET + hash)
   • Global metrics (counters)

✅ Hit retrieval & display:
   • GET /intent/hits endpoint
   • GET /predict/stats endpoint
   • Status line integration
   • E2E test coverage

✅ Autotune integration:
   • Hit decay (~10% per cycle)
   • Domain tier promotion/demotion based on hits
   • Orphan prioritization for learning

❌ Hit attribution (file → domain)
❌ Per-user hit patterns
❌ Real-time hit velocity metrics

⚠️  Partial/Async:
   • Grep hit increments lag ~100ms
   • Prediction hits require manual setup
   • Errors caught silently

REDIS KEY STRUCTURE

DOMAIN HITS:
  aoa:{project}:domain:{name}:meta
  Fields: {hits, total_hits, last_hit_at, tier, state, ...}

TERM HITS:
  aoa:{project}:term_hits
  Hash: {term_name: count, ...}

KEYWORD HITS:
  aoa:{project}:keyword_hits
  Hash: {keyword: count, ...}

ORPHAN HITS:
  aoa:{project}:orphan_hits
  Hash: {tag: count, ...}

PREDICTION HITS:
  aoa:rolling:data:{prediction_key}
  Hash: {hit: '1'|'0'|'', session_id, timestamp, predicted_files}

METRICS:
  aoa:metrics:hits (global counter)
  aoa:metrics:misses (global counter)
  aoa:{project_id}:metrics:hits (per-project)
  aoa:{project_id}:metrics:misses (per-project)

ARCHITECTURE SUMMARY

Three-Tier Hit Tracking System:

TIER 1: Domain/Term/Keyword Hits (Persistent Learning)
  ├─ Updated: Every grep, every intent match
  ├─ Storage: Redis hashes (permanent)
  ├─ Decay: ~10% per autotune cycle
  └─ Purpose: Track semantic element usage for domain curation

TIER 2: Prediction Hits (Rolling 24h Window)
  ├─ Updated: When user reads files
  ├─ Storage: Redis ZSET + rolling data hashes
  ├─ Metric: Hit@5 = (predictions with ≥1 match) / (total)
  ├─ Current: ~82% accuracy
  └─ Purpose: Measure prediction accuracy

TIER 3: Orphan Hits (Pre-Domain Tracking)
  ├─ Updated: When unmatched tags appear
  ├─ Storage: Redis hash (permanent)
  ├─ Purpose: Signal high-demand domains to create

All backed by Redis pipelines (atomic operations).
Mostly async to keep search responsive.
